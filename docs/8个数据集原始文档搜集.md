## 一、现有数据集资源

### 1.1 核电领域机器学习数据集

目前**尚未发现专门针对"核应急问答"的开源大模型训练数据集**，但存在以下相关技术数据集：

| 数据集名称 | 类型 | 说明 | 来源 |
|-----------|------|------|------|
| **NPPAD** | 时序数据 | 核电厂事故数据集，基于PCTRAN模拟器生成，覆盖压水堆常见事故类型 | [Nature Scientific Data](https://www.nature.com/articles/s41597-022-01879-1) |
| **MELCOR模拟数据** | 仿真数据 | 基于福岛事故的MELCOR模拟数据，用于训练LSTM网络预测事故进程 | [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S1738573324000378) |
| **LOCA/SGTR事故数据** | 仿真数据 | 冷却剂丧失事故和蒸汽发生器管道破裂事故模拟数据 | [ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S014919702200138X) |

**结论**：核应急问答数据集需要自行构建，可参考上述技术数据集的构建方法。

### 1.2 核领域专用语言模型与基准数据集

随着LLM在核领域的应用探索，已有若干专门项目值得关注和参考：

| 项目/数据集 | 类型 | 说明 | 来源/论文 |
|------------|------|------|----------|
| **CurieLM** | 微调框架 | 专为核领域LLM应用设计，包含专家验证的高质量数据集构建框架 | [EPJ Conferences](https://www.epj-conferences.org/) |
| **NukeBERT** | 预训练模型 | 基于BERT架构，专门针对核科学领域任务进行预训练 | ResearchGate |
| **NukeLM** | 预训练模型 | 使用美国能源部OSTI数据库的150万篇摘要进行预训练 | [ResearchGate](https://www.researchgate.net/) |
| **NuclearQA** | 评测基准 | 100个专家设计的问题，用于评估LLM在核领域的理解能力 | [arXiv](https://arxiv.org/) |
| **IAEA Safety Glossary** | 结构化知识库 | 核安全术语标准定义，可通过"文本化"转换为微调数据 | [IAEA官网](https://www.iaea.org/resources/safety-standards/safety-glossary) |

## 三、文档搜集工作流程

### 3.1 推荐搜集顺序

```
第一批（官方权威文档）
├── 白皮书、法律法规、国家预案
├── IAEA安全标准（GSR Part 7等）
└── 国家标准（GB/EJ系列）

第二批（技术文档）
├── 监测技术规范
├── 医学应急指南
└── IAEA技术报告

第三批（案例与科普）
├── 事故调查报告
├── 官方科普资料
└── 学术论文
```

### 3.2 文档格式建议

| 格式 | 适用场景 |
|-----|---------|
| PDF | 法规标准、IAEA报告（保留原始格式） |
| Word/TXT | 需要编辑的文档 |
| Markdown | 科普类、可结构化的内容 |

### 3.3 文档处理流程

```
原始文档 → 格式转换 → 文本清洗 → 分块处理 → 问题生成 → 答案生成 → 数据集导出
    ↓
 NuCorpus工具处理
```

## 五、版权与使用说明

### 5.1 可自由使用的资源

- 政府公开文件（白皮书、法律法规、预案）
- IAEA公开出版物
- 官方科普资料

### 5.2 需注意版权的资源

- 学术期刊论文（需获取授权或仅供研究使用）
- 商业出版的教材书籍
- 部分技术标准（需购买正版）

### 5.3 数据集发布建议

如需公开发布数据集，建议：
1. 优先使用政府公开文档
2. 学术论文内容需改写，避免直接复制
3. 标注数据来源，尊重原作者权益

## 九、合成数据生成策略

### 9.1 为什么需要合成数据

核应急领域面临以下数据挑战，使得合成数据生成成为必要手段：

1. **数据稀缺性**：核事故案例数量有限，真实数据难以获取
2. **隐私与安全**：部分数据涉及国家安全或商业机密
3. **专业性极高**：需要领域专家验证数据质量
4. **数据多样性不足**：现有数据覆盖场景有限

### 9.2 合成数据生成方法

#### 方法一：基于知识库的文本化（Textification）

将结构化知识（如术语表、本体）转换为自然语言问答对：

```
输入：IAEA Safety Glossary 中的术语定义
输出：
Q: 什么是核应急？
A: 根据IAEA定义，核应急是指由于核链式反应或放射性物质衰变产生的能量，
   以及辐射照射而发生或预计发生危险的应急状态，需要立即采取行动
   以减轻对人类生命、健康、财产或环境的危害。
```

#### 方法二：基于文档的问答对生成

使用NuCorpus工具自动从原始文档生成问答对：

```
文档片段 → 文本分块 → 问题生成 → 答案生成（含CoT推理） → 质量评估 → 导出
```

#### 方法三：情景模拟数据生成

基于核应急场景设计模板，生成多样化训练数据：

| 场景类型 | 示例 |
|---------|------|
| 事故诊断 | "反应堆冷却剂压力异常下降，可能原因是什么？" |
| 应急响应 | "发生LOCA事故时，操作员应该首先采取什么措施？" |
| 公众防护 | "当收到核事故警报时，普通居民应该如何自我防护？" |
| 剂量评估 | "工作人员在现场停留2小时后，预估受到的辐射剂量是多少？" |

### 9.3 合成数据质量保证

- **专家审核**：所有生成的数据需经核应急领域专家验证
- **事实核查**：与权威文献交叉验证，防止幻觉
- **多样性检查**：确保问题类型、难度、场景的多样性
- **一致性验证**：答案之间不能相互矛盾

## 十一、数据质量控制与标注流程

### 11.1 质量控制维度

| 维度 | 检查内容 | 工具/方法 |
|-----|---------|----------|
| **准确性** | 事实是否正确、是否与权威来源一致 | 专家审核、交叉验证 |
| **完整性** | 答案是否完整覆盖问题 | 人工审核 |
| **一致性** | 不同QA对之间是否矛盾 | NLP相似度检测 |
| **专业性** | 是否符合核应急领域规范用语 | 术语表匹配 |
| **安全性** | 是否包含敏感或有害信息 | 安全审核 |

### 11.2 标注流程建议

```
1. 初审：自动检测明显错误（格式、编码、长度）
     ↓
2. 一审：核应急专业人员审核技术准确性
     ↓
3. 二审：语言质量审核（表达清晰、逻辑通顺）
     ↓
4. 抽检：随机抽取10-20%进行交叉验证
     ↓
5. 归档：记录来源、审核人、修改历史
```

### 11.3 数据标注规范

为保证数据质量一致性，建议遵循以下规范：

- **问题类型标签**：`[事实型]`、`[解释型]`、`[程序型]`、`[判断型]`
- **难度分级**：1-5级（1=基础科普，5=专家级）
- **来源标注**：标注原始文档出处和页码
- **领域标签**：参照8个数据集分类进行标注
